# Technical Guidelines

## Backend
Du schreibst PHP Code nach Nextcloud-Standards. Du nutzt das Backend von Nextcloud statt eigener Lösungen. Du nutzt OCP\IL10N für alle sichtbaren Strings. Du registrierst die App über Application.php mit IBootstrap. app.php entfällt. Controller nutzen Attribute wie #[NoAdminRequired] oder #[NoCSRFRequired]. Die App muss nur mit dem aktuellen Nextcloud Version 32 kompatibel sein. Du verwendest kein autoload, da dies von Nextcloud übernommen wird. Datenbanktabellen werden mittels Migrationen angelegt. database.xml entfällt. Du registrierst die Navigation in der info.xml. Du registrierst alle Routen in appinfo/routes.php. Nutze Tabellennamen mit maximum 23 stellen um mit allen Datenbanktypen kompatibel zu sein. Es wird der gesamte CRUD Datenfluss von Endpunkt bis Controller, Service, Entity und Mapper implementiert. Du benutzt lowerCamelCase und keine "_". In Entities werden keine getter und setter for die attribute erzeugt, da diese automatisch generiert werden. Entities implementieren jsonSerialize.

## Frontend
Du setzt auf Vanilla JS. Du hältst dich an die Vorgaben von Nextcloud. Du nutzt kein webpack. Du schreibst ES6. Du strukturierst deinen Code über Modul-Pattern und Namespaces mit Untermodulen. Du holst Inhalte per AJAX und generierst sie im Client. Du nutzt eigene Contoller Endpoints ohne OCS. Du nutzt die t() API von Nextcloud für alle sichtbaren Strings. Du legst keine Sprachdateien an. Alle AJAX Requests müssen den 'OCS-APIREQUEST': 'true' header senden. Du nutzt keine externen Composer Packete. Die DOM Hierarchie der Hauptseite besteht aus 3 DIVs mit den IDs für app-navigation, app-content, app-sidebar. Navigations Items werden in <ul> listen unter app-navigation dargestellt. Du benutzt lowerCamelCase und keine "_".
